# 离线日出日落缓存系统说明

## 📋 功能概述

## 🔄 工作流程

### 1. 首次访问（在线）
```
页面加载
  ↓
尝试读取缓存（无数据）
  ↓
使用简化判断（6:00-18:00）
  ↓
获取API数据
  ↓
保存到 localStorage
  ↓
使用精确数据更新显示
```

### 2. 再次访问（在线）
```
页面加载
  ↓
读取缓存数据（有数据）
  ↓
使用缓存数据显示
  ↓
获取API数据（更新）
  ↓
更新 localStorage
  ↓
使用最新数据
```

### 3. 离线访问
```
页面加载
  ↓
读取缓存数据
  ↓
缓存存在?
  ├─ 是 → 使用缓存数据（即使过期7天内仍可用）
  └─ 否 → 使用简化判断（6:00-18:00）
```

## 🛠️ API 函数说明

### 保存缓存
```javascript
saveSunriseSunsetData(cityKey, sunrise, sunset)
```
- 自动保存日出日落数据到 localStorage
- 记录保存日期和时间戳

### 读取缓存
```javascript
loadSunriseSunsetData(cityKey)
```
- 返回缓存数据（如果存在）
- 检查数据年龄（超过7天会警告但仍可用）

### 清除缓存
```javascript
clearSunriseSunsetCache()
```
- 清除所有缓存数据（调试用）

### 获取缓存城市
```javascript
getCachedCities()
```
- 返回已缓存数据的城市列表

## 📅 缓存有效期

- **推荐更新周期**: 每天
- **过期提醒**: 7天
- **仍可使用**: 即使过期也能用于离线判断
- **原因**: 日出日落时间变化缓慢，7天内误差可接受

## 🎯 精确度说明

### 在线模式（最精确）
- 考虑季节变化
- 考虑地理位置（经纬度）
- 实时更新

### 缓存模式（较精确）
- 使用最近的日出日落数据
- 1-7天内误差：约 ±2-10分钟
- 对于主题切换完全可接受

### 简化模式（基本精确）
- 固定 6:00-18:00
- 不考虑季节和位置
- 仅作为最后备选方案

## 📱 使用建议

1. **联网时首次访问**: 系统会自动获取并缓存所有城市的数据
2. **定期联网**: 建议每周至少联网一次，更新缓存
3. **长期离线**: 可使用数周，但精确度会逐渐降低
4. **清除缓存**: 如果发现数据异常，可清除缓存重新获取

## ⚠️ 注意事项

1. 浏览器隐私模式可能不保存 localStorage
2. 清除浏览器数据会删除缓存
3. 不同浏览器的 localStorage 是独立的
4. 缓存数据约占用几KB存储空间
